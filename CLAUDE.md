# Karaoke King

Browser-based party karaoke scoring game. React 19 + TypeScript + Vite, deployed to GitHub Pages.

## Tech Stack
- React 19, TypeScript (strict), Vite
- Tailwind CSS v3 + shadcn/ui
- Zustand v5 — game state
- Web Audio API — microphone input and scoring
- React Router v7
- GitHub Actions → GitHub Pages deployment

## Key Directories
- `src/features/` — feature modules (game flow, scoring, audio, etc.)
- `src/data/songs/` — song catalog (`catalog.json`, `category-songs.json`)
- `src/data/coaching/` — fallback coaching cue generator (runtime, for songs without pre-generated cues)
- `src/store/` — Zustand stores
- `src/types/` — shared TypeScript types
- `public/song-data/` — per-song UltraStar data (JSON, one file per song); each file includes a `coachingCues` array

## Development
```bash
npm install
npm run dev       # dev server
npm run build     # production build
npm run preview   # preview build locally
```

## Adding Songs
Songs come from the `../music-metadata-extractor` companion tool. After running extraction and calibration:
```bash
npm run merge-songs -- /path/to/music-metadata-extractor/results/timestamp-dir
```
This merges catalog entries, category assignments, and song-data JSON files. Review with `git diff` before committing.

## Coaching Cues
Each song JSON in `public/song-data/` contains a `coachingCues` array generated by the calibration tool. `useCoachingCues` reads these pre-generated cues directly from `extractedData.coachingCues`. If a song has no pre-generated cues, `src/data/coaching/index.ts` provides a simpler runtime fallback.

To regenerate cues for all existing songs (e.g. after bulk gap re-calibration):
```bash
cd ../music-metadata-extractor
npm run generate-coaching -- ../karaoke-king/public/song-data
```
